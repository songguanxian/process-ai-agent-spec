# 工序 AI 助手 PoC 说明

这是一次性功能原型，用于回答：

> 工序管控系统 AI 助手在 Web/App 中如何承载统一会话、任务节点、结果卡片、OCR 确认、报表简报、固定确认和审计状态？

## 运行方式

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File prototype\ai-assistant-poc\Start-PoC.ps1
```

打开：

```text
http://127.0.0.1:5177/prototype/ai-assistant-poc/?variant=A
```

## 变体

- `A`：Web 业务驾驶舱 + 右侧 AI 侧边栏。
- `B`：App 现场作业模式，突出拍照、语音、常用指令和移动端确认。
- `C`：运营工作台模式，突出任务节点、审计、配置版本和管理视角。

## 覆盖范围

- 覆盖 PRD 与 AI Coding 规格中的 9 个用户故事：操作指南、验收申请草稿、语音检查记录、白板照片识别、项目简报、集团四级简报、责任登记卡 OCR、字段模板配置、审计与失败记录。
- 覆盖 7 类角色：现场作业人员、项目管理人员、集团/直属单位管理人员、监管负责人、业务配置管理员、系统管理员、AI/供应商运维。
- 支持角色切换，点击无权限场景时展示权限拦截和兜底路径。
- 支持任务节点、SSE 事件流、OCR 批次、报表快照、确认动作和审计轨迹的内存状态展示。
- 使用 `work/extracted-docx-images` 中的 Web/App 参考截图作为视觉和信息密度参照。

## 原型约束

- 不连接真实后端。
- 不持久化数据。
- 所有状态保存在浏览器内存。
- 通过底部切换器或键盘左右方向键切换变体。
- 评审完成后应删除或吸收为正式设计，不应直接作为生产代码使用。
